DML (Data Manipulation Language)
===============================

-> Used to manipulate data inside a database
-> Handles user actions (insert, read, update, delete)


INSERT
======

-> Used to insert new data into a table
-> Two modes: explicit (recommended) and implicit

Explicit (recommended):
INSERT INTO utenti (nome, email, eta)
VALUES ('Mario', 'mario@email.com', 25);

Implicit (not recommended):
INSERT INTO utenti
VALUES ('Mario', 'mario@email.com', 25);

NOTE:
-> Always prefer explicit to avoid errors if column order changes


UPDATE
======

-> Used to modify existing data
-> Can be dangerous if used without WHERE

Example:
UPDATE utenti
SET eta = 26
WHERE nome = 'Mario';

WARNING:
UPDATE utenti SET eta = 26;
-> This updates ALL rows

Best practices:
-> Always use WHERE
-> Test with SELECT before UPDATE


DELETE
======

-> Used to remove data from a table

Example:
DELETE FROM utenti
WHERE nome = 'Mario';

WARNING:
DELETE FROM utenti;
-> Deletes ALL rows

Best practices:
-> Check with SELECT first
-> Be sure about conditions


SELECT
======

-> Used to retrieve data from database

Example:
SELECT nome, email
FROM utenti
WHERE eta > 18;

Main parts:
-> SELECT = columns
-> FROM   = table
-> WHERE  = filter rows


JOIN
====

-> Used to combine data from multiple tables

Example:
SELECT utenti.nome, ordini.id_ordine
FROM utenti
JOIN ordini ON utenti.id = ordini.utente_id;

Types:
-> INNER JOIN = only matching rows
-> LEFT JOIN  = all from left table
-> RIGHT JOIN = all from right table


IN
==

-> Checks if value exists in a list

Example:
SELECT *
FROM utenti
WHERE eta IN (18, 25, 30);


BETWEEN
=======

-> Selects a range of values

Example:
SELECT *
FROM utenti
WHERE eta BETWEEN 18 AND 30;


LIKE
====

-> Pattern search

Example:
SELECT *
FROM utenti
WHERE nome LIKE 'Mar%';

Wildcards:
-> % = any sequence of characters
-> _ = single character


ORDER BY
========

-> Sorts results

Example:
SELECT *
FROM utenti
ORDER BY nome ASC;

Options:
-> ASC  = ascending (default)
-> DESC = descending


AS (Alias)
==========

-> Renames columns or tables temporarily

Example:
SELECT nome AS nome_utente
FROM utenti;


EXTRA
=====

LIMIT
-----
-> Limits number of results

SELECT *
FROM utenti
LIMIT 5;


DISTINCT
--------
-> Removes duplicates

SELECT DISTINCT eta
FROM utenti;


AND / OR
--------

-> Multiple conditions

Example:
SELECT *
FROM utenti
WHERE eta > 18 AND nome LIKE 'M%';
